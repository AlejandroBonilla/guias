# Deploy

Cuando nuestro desarrollo concluye 😌, es momento de probar nuestra aplicación , como lo haría el usuario final. Esto significa hacer un build de ella para llevarla a producción. Para esto disponemos de algunas herramientas que nos facilitan este proceso.

# Despliegue en Producción

Este proceso es importante ya que nos permite optimizar nuestra aplicación , esto significa que al hacer build nos ayudara a comprimir/empaquetar al máximo los archivos de nuestro proyecto, desechando aquello que no es necesario, para que asi la carga de la aplicación en el navegador sea lo mas rápida posible.

# Deploy aplicación creada con VUE/CLI

Si nuestra aplicacion fue instalada con vue/cli , deberemos correr el siguiente comando en el terminal.

npm run build

Este proceso al terminar creara una carpeta de nombre dist la cual contiene todo nuestro proyecto pero optimizado , sin opciones de desarrollo.

# Instalación serve

Para poder servir nuestra aplicación recién construida podríamos utilizar la librería serve , esta nos permite servir sitios estáticos, SPA, o solo un archivo estático.

para instalarla podriamos ejectuar el siguiente comando.

npm install -g serve

# Ejecutando serve

Unas ves instalado serve, debemos posicionarnos en la raíz de nuestro proyecto si haces ls , deberías poder ver los archivos de vue , y la carpeta dist.

ejecutamos

serve -s dist

Lo que hare sera levantar nuestra aplicación como si un servidor se tratase

Una ves agregadas las dependencias debemos instalarlas , desde la raíz del proyecto ejecutamos

npm install

ahora debemos crear los archivos de configuración de webpack , para esto los vamos a estructurar en 4 archivos

  • webpack.config.js : Sera el archivo principal de nuestra configuración que gestionara que instrucciones se ejecutaran para el empaquetado.
  • webpack.config.base.js: Sera el archivo que contrada todas las configuraciones de empaquetado que sean común tanto para una compilación de archivos que desean destinados a un modo desarrollo como a un modo producción.
  • webpack.config.dev.js : Sera el archivo que contendrá las configuración para empaquetar una aplicación que corra en modo desarrollo.
  • webpack.config.prod.js: sera el archivo que contendrá las configuraciones propias para empaquetar una aplicación que corra en modo producción.

webpack.config.js

const merge = require("webpack-merge");
const baseConfigGenerator = require("./webpack.config.base.js");
const prodConfig = require("./webpack.config.prod.js");
const devConfig = require("./webpack.config.dev.js");
function webpackEnviromentSelector(env) {
  let config;
  if (env.production) config = prodConfig;
  if (env.development) config = devConfig;
  const baseConfig = baseConfigGenerator(env);
  return merge(baseConfig, config);
}
module.exports = webpackEnviromentSelector;

webpack.config.base.js

var path = require('path')
var webpack = require('webpack')
const VueLoaderPlugin = require('vue-loader/lib/plugin') 
const HTMLWebpackPlugin = require('html-webpack-plugin')
function webpackConfigGenerator(env) { const sourcemaps = !!env.development; const webpackInitConfig = {
resolve: {
extensions: ['*', '.js', '.vue', '.json']
},
entry: './src/main.js', output: {
path: path.resolve(__dirname, './dist'), publicPath: '/',
filename: 'build.js'
}, module: {
rules: [ {
test: /\.css$/, use: [

{loader: 'vue-style-loader', options: { sourceMap:sourcemaps }},

{loader: 'css-loader', options: { sourceMap: sourcemaps}}],
exclude: /node_modules/, },
{
test: /\.vue$/,
loader: 'vue-loader',
exclude: /node_modules/, },
{
test: /\.js$/,
loader: 'babel-loader',
exclude: /node_modules/ },
{
test: /\.(png|jpg|gif|svg)$/, loader: 'file-loader', options: {
name: '[name].[ext]?[hash]' }
} ]
}, plugins: [
          new VueLoaderPlugin(),
          new HTMLWebpackPlugin({
filename: './index.html',
template: 'index.html', })
] };
  return webpackInitConfig;
}
module.exports = webpackConfigGenerator;

webpack.config.dev.js

const webpackConfig = {
  mode: 'development', devtool: 'inline-source-map',
  };
  module.exports = webpackConfig;

webpack.config.prod.js

const OptimizeCssAssetsPlugin = require('optimize-css-assets-webpack-plugin'); const webpackConfig = {
  mode: 'production',
  plugins: [
    new OptimizeCssAssetsPlugin()
  ],
 };
 module.exports = webpackConfig;

con estas configuraciones ya podriamos ejecutar los camandos tanto para desarrollo como para produccion.

modo desarrollo

npm run webpack-dev

modo producción

npm run webpack-prod
