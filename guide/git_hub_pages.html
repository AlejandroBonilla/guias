# Github-pages Deploy

# Crear repositorio en github

Para hacer un deploy de manera simple , crearemos un repositorio nuevo en github para dejar nuestro deploy y luego dejarlo disponible con la ayuda de github pages

Solo agregaremos el nombre del repositorio en ( repository name ), dejaremos el repo en (public) y damos click en el botón verde create repository

# Obtener ruta repo

la parte importante es guardar la ruta de nuestro repo en mi ejemplo uso "ssh" pero con "https" no debería haber problema.

git@github.com:AlejandroBonilla/app_name.git

# Creando bash para deploy

En nuestro proyecto de vue.js en la raiz debemos crear un archivo , para este ejemplo usare deploy.sh como nombre del archivo.

dentro de este archivo bash podremos escribir las instrucciones que normalmente hacemos por el terminal pero este archivo nos permite ejecutar un grupo de instrucciones.

# deploy.sh

#!/usr/bin/env sh

# abort on errors
set -e

# build
npm run build

# navigation to output directory
cd dist
cp index.html 404.html
# git instrucction

git init 
git add -A
git commit -m "deploy"


# Para desplegar nuestra pagina en gh-pages  https://<USERNAME>.github.io/<REPO>
# !importante ! reemplazar por su propio git la siguiente linea donde corresponda

git push -f git@github.com:AlejandroBonilla/app_name.git master

cd -

si nos fijamos bien en la penúltima linea del archivo estamos realizando el push forzado al repositorio antes creado. Este archivo se puede ejecutar las veces que deseemos , esto actualizara automáticamente nuestro repo, dejando un único commit.

# modificando vue.config.js

Para que nuestro proyecto se despliegue de manera correcta en github pages 😏 , vamos a editar el archivo de configuración de vue.

Vamos a agregar la siguiente lineas dentro del objeto *module.exportes

  publicPath: process.env.NODE_ENV === 'production'
    ? '/app_name/'
    : '/'

donde /app_name es el nombre tu proyecto en github

el archivo debe quedar de la siguiente forma

module.exports = {
  //more config,
  publicPath: process.env.NODE_ENV === 'production'
    ? '/app_name/'
    : '/'
}

Cabe destacar si el archivo tiene mas configuraciones por ejemplo vuetify , debemos agrega una coma ya que estamos dentro de un objeto y añadirlo las lineas anteriores debajo 👍.

# Ejecutando bash

Si hemos seguido todas las instrucciones , deberías poder ejecutar nuestro script bash para realizar nuestro deploy. vamos al terminar y debemos chekear que estemos en a raíz del proyecto

sh deploy.sh

Este comando ejecutara un build de nuestro proyecto , posterior a eso hará un push al repositorio anteriormente creado.

# Habilitar gb-pages

Una vez hayamos desplegado nuestro proyecto y no hemos tenido errores 💥 😫.

Podemos dirigirnos a github y buscar nuestro con el commit "deploy"

Estando en el repositorio buscaremos la pestaña settings

iremos al panel izquierdo y buscaremos Pages

seleccionaremos la rama master

y le damos click en save

Esto nos generara el link de acceso a nuestro pagina

El link podría tardar unos minutos en habilitarse 👀.
